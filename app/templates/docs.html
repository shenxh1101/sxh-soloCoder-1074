API 文档

OAuth2.0 端点

GET /oauth/authorize

授权端点 - 获取授权码（Authorization Code）

请求参数：

  • response_type - 必须，值为 code
  • client_id - 必须，客户端ID
  • redirect_uri - 必须，回调地址
  • scope - 可选，权限范围，空格分隔
  • state - 推荐，用于防止CSRF攻击

示例：

GET /oauth/authorize?response_type=code&client_id=xxx&redirect_uri=http://localhost/callback&scope=read:user%20write:data&state=random123

POST /oauth/token

令牌端点 - 换取访问令牌

认证方式：

HTTP Basic Auth (client_id:client_secret)

请求参数（application/x-www-form-urlencoded）：

  • grant_type - 必须，authorization_code 或 client_credentials 或 refresh_token
  • code - 授权码模式需要，授权端点返回的code
  • redirect_uri - 授权码模式需要，与授权请求一致
  • refresh_token - 刷新令牌模式需要
  • scope - 客户端凭证模式可选

示例（客户端凭证模式）：

curl -X POST http://localhost:5000/oauth/token \ -u "client_id:client_secret" \ -d "grant_type=client_credentials" \ -d "scope=read:user write:data"

POST /oauth/introspect

令牌验证端点 - 供资源服务器验证令牌

认证方式：

HTTP Basic Auth (client_id:client_secret)

请求参数：

  • token - 必须，要验证的访问令牌
  • token_type_hint - 可选，access_token 或 refresh_token

响应示例：

{ "active": true, "scope": "read:user write:data", "client_id": "xxx", "username": "testuser", "token_type": "Bearer", "exp": 1234567890, "iat": 1234564290 }

POST /oauth/revoke

令牌撤销端点

认证方式：

HTTP Basic Auth (client_id:client_secret)

请求参数：

  • token - 必须，要撤销的令牌

管理 API

GET /api/clients

获取所有客户端列表

POST /api/clients

创建新客户端

请求体：

{ "name": "My App", "description": "My test application", "redirect_uris": ["http://localhost/callback"], "grant_types": ["authorization_code", "client_credentials"], "token_format": "jwt", "token_expire_seconds": 3600, "require_consent": true }

GET /api/clients/{id}

获取客户端详情

PUT /api/clients/{id}

更新客户端

DELETE /api/clients/{id}

删除客户端（同时删除相关令牌）

GET /api/export/clients

导出客户端配置为JSON

GET /api/export/tokens

导出令牌数据为JSON

GET /api/export/all

导出所有数据（客户端、令牌、Scope、日志等）为JSON

快速开始

1. 创建客户端

访问 创建客户端 页面，填写表单创建新的OAuth客户端。

2. 测试授权码模式

  1. 构造授权URL，在浏览器中访问
  2. 输入任意用户名登录（例如：testuser）
  3. 点击"允许授权"同意授权
  4. 复制回调URL中的 code 参数
  5. 使用code调用令牌端点换取访问令牌

3. 测试客户端凭证模式

直接使用curl或Postman调用令牌端点：

curl -X POST http://localhost:5000/oauth/token \ -u "your_client_id:your_client_secret" \ -d "grant_type=client_credentials"

4. 验证令牌

curl -X POST http://localhost:5000/oauth/introspect \ -u "your_client_id:your_client_secret" \ -d "token=your_access_token"

5. 模拟错误场景

访问 模拟错误管理 页面，启用需要测试的错误场景。
